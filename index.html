My name is Filip Sedlák and I live in Prague, in the middle of Europe.

I'm a developer and I enjoy it. :-) Because of existing codebases and collaboration with other developers, I use mostly Java. However, I'd like better to use Scala or Clojure. For smaller projects and command-line tools, I'm using often Python. Continuously, I try to improve my Haskell skills.

Part of my work involves front-end development of web applications. I like the approach of single-page Javascript application supported by REST web services with well documented interface. AngularJS really helps here.

I put emphasis on discussing the ideas and approaches with other people rather than just developing programs. This includes both customers and fellow-programmers.

I'd like to move on to design even more complex systems. If you're working on an interesting problem, drop me a line.

My current work

Instant JChem

For ChemAxon, the world leader in cheminformatics, I'm contributing to a chemically inteligent tool for managing data. Instant JChem connects to existing databases at world's top pharmaceutial companies and allows to search, save, combine and upload data. It also provides access to broad portfolio of cheminformatics functionality provided by ChemAxon.

Currently, with a small team, we focus on bringing the existing tools to a web platform and to better connect and search the existing data. This involves a lot of communication and also travel. I found out that sometimes it is more efficient to travel across Europe than to have a Skype call.

My previous work

XMR GUI

Graphical user interface for XMR software. That program performs a simulation of catalytic converters [1]. The GUI is made with GWT to be able to run simulations remotely on different servers without the need of client installation. This is very good for quick introduction of new students to this software's usage. I was a member of team of three developers under the excellent lead of Míra Šimek.

Space cartographer

Development of dynamic web application for navigation across the chemical space. The chemical compounds can be put into a mutlidimensionary space and reports from this space can give useful insights during drug discovery process. For more info, see the slides from thesis defense (in Czech).

The challenging part here was to integrate existing Python, Java and R code in a web application. Did you know the CDK's descriptor calculators are not at all thread safe and you can't interrupt the computation if it takes too long?

This was my MSc thesis under the lead of insightful assoc. prof. Dan Svozil (head of cheminformatics laboratory @ ICT).

Knime reporting for IMG

Before the KNIME reporting extension came out, we started a project to create reports from the KNIME workflows. This got to a stage where several nodes were developed which were able to produce an HTML page with data from the workflow.

However, since the beginning, KNIME was used in that lab mostly as an inerface to R scripts. It did not gain much popularity among the scientists there, so it was replaced with combination of Python and R.

Structural analysis of DNA-protein interface

As a BS thesis, I processed publicly available crystallography data to find statistically significant patterns in binding between DNA and proteins, including H and water-mediated bonds.

Call centre apps

My first real contract was with Prague call centre Eficia to develop web applications used by their phone operators to enter and look up data about customers. LAMP stack and Javascript using specific IE features were the tools of the day.
